## 修复毛选卡片翻牌下移 bug
- 问题：翻牌后卡片正面内容溢出到卡片外部，"再抽一张"按钮被盖住
- 根本原因：
  1. `.card-back` 有重复的 `position` 声明（absolute + relative 写了两次）
  2. `.quote-card.hidden` 用 `display:none` 破坏了 3D 翻转的 DOM 结构
- 修复内容：
  - 清理 `.card-back` 的重复 position，改为单一 `position:absolute` + `overflow:hidden`
  - `.quote-card.hidden` 改用 `visibility:hidden`（保持 DOM 占位，3D 翻转正常）
  - `transition` 从 `.card-container.flipped` 移到 `.card-container`，确保回翻也有动画
  - `card-display-area` 从 `min-height` 改为固定 `height:480px` 防止撑开
  - JS `performDraw` 逻辑：翻回时等待 900ms（完整动画时长）再重新抽牌

## 修复答案之书点击无反应（二次修复，2026-03-29）
- 根本原因：`querySelector('.book-hint')` 是全局查询，在页面未激活时可能返回null或找到其他页面元素，导致函数报错中断
- 修复方案：
  - `openAnswerBook()` 和 `resetAnswerBook()` 统一改为 `page.querySelector('.book-hint')` 用页面作用域查询
  - 添加防重复点击：`book.onclick = null`，在重置完成后再恢复 `book.onclick = openAnswerBook`
  - 答案显示逻辑简化：不再嵌套多层 setTimeout，改为两步：600ms后隐藏书本 → 400ms后显示答案并淡入

## 修复塔罗牌页面空白问题
- 问题：选牌区域隐藏后页面出现大量空白
- 根本原因：`card-selection` 使用 `display:none` 隐藏导致布局塌陷
- 修复内容：
  - CSS 添加 `#card-selection` 固定高度 `min-height: 400px`
  - `.hidden` 类改用 `visibility:hidden` + `opacity:0` 保持布局占位
  - JS `resetTarotState()` 和 `showSpreadResult()` 更新为使用 visibility 控制显示/隐藏
